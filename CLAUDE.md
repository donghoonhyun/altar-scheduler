## HTML 자동번역 차단 규칙 (필수 엄수)

`index.html`은 **반드시 아래 세 가지를 모두 적용**해야 한다.
전례 용어, 성인명, 기도문 등이 자동번역으로 오역될 수 있으므로 절대 허용하지 않는다.

```html
<!-- html 태그 -->
<html lang="ko" translate="no" class="notranslate">

<!-- head 내 meta 태그 -->
<meta name="google" content="notranslate" />
```

| 방법 | 차단 대상 |
| --- | --- |
| `translate="no"` | HTML Living Standard — 모든 브라우저 내장 자동번역 |
| `class="notranslate"` | Google 번역 확장 프로그램 |
| `<meta name="google" content="notranslate">` | Google 웹 번역 서비스 / 크롤러 |

**새 앱·페이지 추가 시에도 반드시 위 세 가지를 모두 적용할 것.**

## Project Overview

**Altar Scheduler (복사 스케줄러)** — A Firebase-based React PWA for managing Catholic church server (altar server) scheduling. It is part of the **Ordo** ecosystem, sharing a single Firebase project (`ordo-eb11a`) with other Ordo apps. SSO is provided by the Ordo main app at `https://ordo.web.app`.

## Common Commands

### Frontend (Vite)

```bash
npm run dev              # Start dev server at localhost:5173
npm run build            # TypeScript compile + Vite bundle to dist/
npm run lint             # ESLint
npm run preview          # Preview production build
```

### Environment Switching

```bash
npm run env:dev          # Switch to development Firebase config
npm run env:prod         # Switch to production Firebase config
npm run env:local        # Switch to local emulator config
```

### Firebase Emulators

```bash
npm run dev:emu          # Vite + Firebase emulators + seed roles (all together)
npm run emulators:start  # Build functions then start emulators
npm run sync:cloud       # Fetch cloud data and import to emulator
```

### Cloud Functions

```bash
npm run build:functions                      # Build altar-scheduler functions
cd functions && npm run build                # Same, from functions/ dir
cd functions && npm run build:watch          # Watch mode
firebase deploy --only functions             # Deploy all functions
firebase deploy --only hosting:ordo-altar   # Deploy hosting only

# Functions 배포 시 로딩 타임아웃 발생하면:
FUNCTIONS_DISCOVERY_TIMEOUT=60000 firebase deploy --only functions
```

### Firestore 데이터 마이그레이션 스크립트

```bash
# 드라이런 (기본): 이동 대상 건수 확인만
```

## Architecture

### Two Function Codebases

| Codebase | Directory | Prefix | Purpose |
| --- | --- | --- | --- |
| `altar-scheduler` | `functions/` | `altar_` | App-specific: mass events, role requests, SMS, daily reminders |
| `ordo-admin` | `functions-ordo-admin/` | `admin_` | Shared: SSO tokens, centralized notifications, AI/crawlers |

Both deploy to the same Firebase project (`ordo-eb11a`) in region `asia-northeast3` (Seoul).

### Firestore Rules

`firebase.json` points rules to `../Ordo/firestore.rules` (a sibling repo). When editing security rules, modify the Ordo project, not this one. Run `npm run rules:merge` and `npm run rules:deploy` from the Ordo project.

### Auth & SSO Flow

`src/App.tsx` handles SSO: it detects an `authtoken` query param injected by the Ordo login page, calls `signInWithCustomToken`, then cleans the URL. The custom token is generated by `admin_generateCrossAppAuthToken` in `functions-ordo-admin/`.

### Role System & Session State

`src/state/session.ts` exports the `useSession()` hook — the single source of truth for auth state. It provides:

- `user` — Firebase Auth user
- `groupRoles` — `Record<serverGroupId, string[]>` where values are `admin`, `planner`, or `server`
- `isSuperAdmin` — boolean from user profile
- `serverGroups` — user's accessible groups with parish info
- `currentServerGroupId` — active server group
- `userInfo` — profile (name, baptismal name, parish)

Role priority for home redirect: **Admin → Planner → Server → SuperAdmin → pending-approval → add-member**.

### Routing

`src/routes/AppRoutes.tsx` uses React Router v7 with role-based redirects. Route access:

- `/superadmin/*` — SuperAdmin only
- `/server-groups/:id/admin/*` — Admin role required
- `/server-groups/:id/mass-events` — Planner role required
- `/survey/:serverGroupId/:yyyymm` — Server (member) role
- The `HomeRedirect` component handles the role-priority redirect logic

### Data Fetching Pattern

TanStack React Query (`@tanstack/react-query`) handles all Firestore data fetching. `useSession()` for auth/role state; custom hooks in `src/hooks/` for feature data (e.g., `useMassEvents`, `useParishes`).

### UI Components

Radix UI primitives + Shadcn-style components live in `src/components/ui/`. Shared layout components (`PremiumHeader`, `DrawerHeader`, `LoadingSpinner`, etc.) live in `src/components/common/`. Tailwind CSS with class-based dark mode (`dark:` prefix) and the Pretendard Korean font.

### PWA & Emulator Notes

- Vite PWA plugin generates service workers. In dev, `src/App.tsx` clears dev-sw/workbox caches on startup.
- Emulator connection is controlled by `isDev` in `src/lib/firebase.ts` (currently hardcoded `false` — toggle manually for emulator work).
- Emulator ports: Auth :9099, Firestore :8080, Functions :5001, Hosting :5000.

### Path Alias

`@/` maps to `src/` in both TypeScript and Vite config.

### Mobile (Capacitor)

Capacitor config (`capacitor.config.ts`) targets Android in `android/`. Build with Vite then sync to Capacitor for native builds.
